## 📌 第二部分：`nvidia-smi` 完全指南

### 2.1 核心功能板块

| 功能类别         | 常用命令示例                                 | 参数选项与作用                                               | 注意事项                          |
| :--------------- | :------------------------------------------- | :----------------------------------------------------------- | :-------------------------------- |
| **基础监控**     | `nvidia-smi`                                 | 无参数：显示静态表格（驱动版本、GPU 利用率、显存、温度、功耗） | 观察状态栏是否有 `ERR!` 或 `N/A`  |
| **动态刷新**     | `nvidia-smi -l 1`                            | `-l, --loop`：指定刷新间隔（秒）                             | 临时监控用，生产环境推荐 `dmon`   |
| **流式数据采集** | `nvidia-smi dmon -s pucvmet -d 1`            | `-s`：选择输出字段（p=功耗，u=利用率，c=时钟，v=电压，m=显存，e=ECC，t=温度） `-d`：采样间隔 | 输出 CSV 格式，适合脚本解析       |
| **深度故障诊断** | `nvidia-smi -q`                              | `-q, --query`：显示所有详细信息 `-d`：指定查看类别（`TEMPERATURE`, `PERFORMANCE`, `ECC` 等） | 使用 `-q -i ` 精准定位            |
| **性能瓶颈分析** | `nvidia-smi -q -d PERFORMANCE`               | 查看 `SW Power Cap`、`HW Slowdown` 等降频原因                | 判断是功耗墙、温度墙还是软件限制  |
| **显存健康检查** | `nvidia-smi -q | grep -A 10 "Retired Pages"` | 查看退役页面数（`Single Bit ECC` / `Double Bit ECC`）        | `Double Bit` 非零即为硬件死刑     |
| **管理操作**     | `nvidia-smi -pm 1`                           | `-pm, --persistence-mode`：开启持久模式，防止驱动频繁卸载    | **服务器必开**，否则易触发 Xid 43 |
| **GPU 软复位**   | `nvidia-smi -r -i  -c 1`                     | `-r, --gpu-reset`：功能级复位 `-c 1`：强制复位模式（用于故障卡） | 对业务卡慎用，会导致进程崩溃      |
| **故障卡隔离**   | `nvidia-smi drain -p 1 -i `                  | `drain`：标记排水，移除出调度池 `-p 1`：同时将 PCIe 链路置为 L2 低功耗 | 比直接 `-r` 更安全，用于硬件隔离  |
| **功耗上限设置** | `nvidia-smi -i  -pl 400`                     | `-pl, --power-limit`：设置功耗墙（单位 W）                   | 误操作会导致性能骤降（如剧本 B）  |
| **拓扑查看**     | `nvidia-smi topo -m`                         | `-m`：显示矩阵拓扑（NVLink/PHB/SYS）                         | 多卡训练必看，确认通信路径最优    |

### 2.2 状态列特殊标记解读

| 显示        | 含义                     | 运维动作                                    |
| :---------- | :----------------------- | :------------------------------------------ |
| **`ERR!`**  | GPU 处于不可恢复错误状态 | 立即提取 `dmesg` 和 `-q` 信息，准备 `drain` |
| **`Drain`** | 已主动隔离，等待移除     | 确认调度器不再分配任务，可安全下电          |
| **`Off`**   | 驱动标记为离线           | 检查 PCIe 链路（Xid 79 常见）               |
| **`P8`**    | 低功耗空闲状态           | 正常，等待任务唤醒                          |
| **`P0`**    | 全速工作状态             | 正常高负载状态                              |

## 📌 第三部分：`dmesg` 完全指南

### 3.1 核心参数与场景

| 使用场景             | 命令示例                         | 参数选项与作用                                         | 注意事项                                   |
| :------------------- | :------------------------------- | :----------------------------------------------------- | :----------------------------------------- |
| **查看最新日志**     | `dmesg -T | tail -20`            | `-T, --ctime`：将时间戳转换为人类可读格式（**必加**）  | 不加 `-T` 只能看到 `[ 15.234567]` 相对时间 |
| **筛选 GPU 相关**    | `dmesg -T | grep -i nvidia`      | `-i`：忽略大小写                                       | 检查驱动加载是否成功                       |
| **精准定位故障卡**   | `dmesg -T | grep -i "1F:00|Xid"` | 结合 PCIe Bus ID 和 Xid 关键字                         | **取证第一步**                             |
| **过滤错误级别**     | `dmesg -l err`                   | `-l, --level`：按日志级别过滤（`err`, `warn`, `info`） | 快速定位红字报错                           |
| **查看驱动加载过程** | `dmesg | grep NVRM`              | NVRM 是 NVIDIA 内核模块的日志前缀                      | 判断驱动版本及 GPU 初始化状态              |

### 3.2 关键 Xid 错误码速查表

| Xid     | 名称                       | 含义                     | 处置建议                        |
| :------ | :------------------------- | :----------------------- | :------------------------------ |
| **48**  | Double Bit ECC Error       | 显存物理损坏（不可纠正） | **立即隔离 RMA**                |
| **79**  | GPU has fallen off the bus | GPU 从 PCIe 总线消失     | 检查 Riser 卡、供电、ASPM 设置  |
| **31**  | GPU Memory Page Fault      | 应用层越界访问或 OOM     | 检查代码 batch size / 显存泄漏  |
| **43**  | GPU Fallback to Bus Idle   | 驱动初始化失败           | 开启 Persistence Mode，检查供电 |
| **92**  | High single-bit ECC rate   | 显存老化前兆             | 预警，计划更换                  |
| **62**  | ECC Page Retirement Failed | 坏块退役失败             | 同 Xid 48，立即 RMA             |
| **45**  | Preemptive Channel Removal | 驱动踢出异常任务         | 通常无害，重启容器即可          |
| **109** | Insufficient Power         | 供电不足                 | 检查电源线缆和 PSU 功率         |

## 📌 第四部分：`lspci` 完全指南

### 4.1 核心参数与场景

| 使用场景         | 命令示例                            | 参数选项与作用                               | 注意事项                                |
| :--------------- | :---------------------------------- | :------------------------------------------- | :-------------------------------------- |
| **确认硬件可见** | `lspci | grep -i nvidia`            | 无参数：列出所有 PCI 设备                    | **装机第一件事**，无输出则检查电源/插槽 |
| **查看详细能力** | `sudo lspci -vvv -s `               | `-vvv`：极详细输出 `-s`：指定设备地址        | 必须加 `sudo` 才能看全寄存器            |
| **带宽协商状态** | `sudo lspci -vvv -s  | grep LnkSta` | 查看 `LnkSta` 字段                           | **诊断“跑得慢”必用**                    |
| **查看设备树**   | `lspci -t`                          | `-t`：树状图显示 PCIe 桥接关系               | 理解多卡拓扑和 NUMA 亲和性              |
| **查看厂商 ID**  | `lspci -nn | grep NVIDIA`           | `-nn`：显示厂商和设备 ID（如 `[10de:20b0]`） | 用于识别未知设备                        |

### 4.2 `LnkSta` 输出解读

| 输出内容         | 正常值          | 异常值                 | 含义                                |
| :--------------- | :-------------- | :--------------------- | :---------------------------------- |
| **Speed**        | `16GT/s` (Gen4) | `2.5GT/s (downgraded)` | 降速到 PCIe 1.0，Riser 卡接触不良   |
| **Width**        | `x16`           | `x8` 或 `x4`           | 通道数减半，检查插槽或 CPU 拆分设置 |
| **`downgraded`** | 无              | 出现                   | 明确指示链路非正常降级              |

## 📌 第五部分：生产环境 GPU 故障排查 SOP（标准作业程序）

> **核心逻辑**：先软后硬，先取证后操作，先隔离后维修。

### 步骤 1：发现异常（巡检）

- **触发条件**：监控告警 `GPU Temp=N/A`、`SM=ERR`、或业务反馈卡慢。
- **第一命令**：`nvidia-smi`
- **观察重点**：状态栏是否有 `ERR!` / `Off` / `Drain`，功耗是否撞墙。

### 步骤 2：现场取证（固定证据）

- **目的**：防止重启后日志丢失，为 RMA 提供依据。
- **动作清单**：
  1. `nvidia-smi -q -i  > gpu_fault.log`
  2. `dmesg -T | grep -i "BusID\|Xid" > gpu_dmesg.log`
  3. `nvidia-smi -q -i  | grep -A 10 "Retired Pages"` 确认显存伤残等级。
  4. `sudo lspci -vvv -s  | grep LnkSta` 确认 PCIe 链路状态。

### 步骤 3：尝试抢救（可选）

- **适用**：仅限 Xid 79 或功耗/温度墙误报。
- **命令**：`nvidia-smi -r -i  -c 1`
- **注意**：对 Xid 48 通常无效。

### 步骤 4：止损隔离（必须执行）

- **命令**：`nvidia-smi drain -p 1 -i `
- **验证**：`nvidia-smi` 状态变为 `Drain`。
- **目的**：防止故障卡 PCIe 错误干扰同机其他健康卡。

### 步骤 5：业务恢复

- **单卡任务**：调度器自动重调度。
- **分布式任务**：修改 `CUDA_VISIBLE_DEVICES` 排除坏卡，重启训练。

### 步骤 6：硬件维修

- **Xid 48/62**：提交 RMA，附上取证日志。
- **Xid 79 + downgraded**：申请停机，清洁 Riser 卡金手指，重新插拔。

## 📌 附录：命令参数速查表（按字母排序）

### `nvidia-smi`

| 短参数 | 长参数               | 作用                                   |
| :----- | :------------------- | :------------------------------------- |
| `-c`   | `--compute-mode`     | 设置计算模式（0=Default, 1=Exclusive） |
| `-d`   | `--display`          | 配合 `-q` 显示特定类别                 |
| `-i`   | `--id`               | 指定 GPU 索引或 UUID                   |
| `-l`   | `--loop`             | 循环刷新                               |
| `-pl`  | `--power-limit`      | 设置功耗上限（W）                      |
| `-pm`  | `--persistence-mode` | 开启持久模式（1=开，0=关）             |
| `-q`   | `--query`            | 显示完整信息                           |
| `-r`   | `--gpu-reset`        | 复位 GPU                               |
| `-c 1` | `--gpu-reset-mode=1` | 强制复位模式                           |

### `dmesg`

| 短参数 | 长参数     | 作用             |
| :----- | :--------- | :--------------- |
| `-T`   | `--ctime`  | 显示人类可读时间 |
| `-l`   | `--level`  | 按日志级别过滤   |
| `-k`   | `--kernel` | 仅显示内核消息   |
| `-H`   | `--human`  | 人性化分页输出   |

### `lspci`

| 短参数 | 长参数             | 作用                      |
| :----- | :----------------- | :------------------------ |
| `-v`   | `--verbose`        | 详细模式（可叠加 `-vvv`） |
| `-s`   | `--slot`           | 指定 PCI 槽位地址         |
| `-t`   | `--tree`           | 树形拓扑显示              |
| `-nn`  | —                  | 显示厂商和设备 ID         |
| `-k`   | `--kernel-drivers` | 显示使用的内核驱动        |
